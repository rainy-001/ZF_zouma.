## 一、Vofa+ 调参配置

### 1.1 架构

```
PC (Vofa+ TCP Server :8086)  ←──TCP──  小车 (TCP Client)
192.168.79.38                            192.168.79.125
```

### 1.2 main.cpp 关键修改

```cpp
#include "my_global.hpp"
#include "config_seting.hpp"   // TCPClient, parseAndUpdateParameter 等

int main()
{
    car_init();  // 完整初始化：电机/IMU/屏幕/摄像头/控制线程/key_scan

    TCPClient client;
    if (!client.connect("192.168.79.38", 8086)) {
        return -1;
    }
    onto_pd_control_enable = 1;

    // car_init() 已启动 key_scan 线程，也已初始化显示屏，不需重复

    while (true) {
        // 接收 Vofa+ 调参命令
        if (client.receiveLine(received, 100)) {
            received.erase(received.find_last_not_of("\r\n") + 1);
            if (received == "WRITE") {
                handleWriteCommand(client);    // 保存参数到文件
            } else if (received == "READ") {
                handleReadCommand();           // 从文件重新加载
            } else {
                parseAndUpdateParameter(received); // 实时更新
            }
        }

        // 上传实时数据
        client.sendFormattedData(
            "Onto_control,yaw,speed_l,speed_r,l_pwm,r_pwm:%f,%f,%f,%f,%d,%d\r\n",
            onto_control, ahrs.getYaw(),
            left_speed, right_speed,
            speed_to_pwm_l, speed_to_pwm_r);

        menu_system.menu_system();
        system_delay_ms(10);
    }
}
```

## 二、遇到的坑与解决

### 2.1 缺少 `config_seting.hpp`

编译报错，`TCPClient`、`parseAndUpdateParameter` 等未定义。
→ 加 `#include "config_seting.hpp"`

### 2.2 key_scan 线程重复启动

`car_init()` 内部已启动 key_scan (my_global.cpp:299-307)，main.cpp 又启动一次。第二次失败导致 `return false` 程序退出。
→ 注释掉 main.cpp 中的 key_scan 启动

### 2.3 car_config.txt 未找到

打印出的 LADRC 参数是代码默认值 (r=150, wc=1, b0=47)，而不是配置文件的正确值 (r=13.3, wc=29, b0=0.83)。
→ 把 car_config.txt 拷贝到小车的 `/home/root/`

### 2.4 TCP 连接失败 (Connection failed)

原因是 PC IP 与代码中不一致。
→ 把 `192.168.43.94` 改成 PC 实际 IP `192.168.79.38`

### 2.5 Broken pipe (手动断电)

连接成功后手动切断小车电源，TCP 自然断开，非代码问题。

## 三、Vofa+ 命令速查

### 实时调参命令

| 命令 | 示例 | 说明 |
|------|------|------|
| `b0:值` | `b0:0.83` | LADRC 系统参数（左右轮同步） |
| `wc:值` | `wc:29` | LADRC 控制器带宽 |
| `w0:值` | `w0:190` | LADRC 观测器带宽 |
| `r:值` | `r:13.3` | LADRC 跟踪速度因子 |
| `h:值` | `h:0.001` | LADRC 积分步长 |
| `kp:值` | `kp:0.02` | 方向 PD 线性比例 |
| `kp2:值` | `kp2:0.017` | 方向 PD 非线性比例 |
| `kd:值` | `kd:1.9` | 方向 PD 微分系数 |
| `limit:值` | `limit:2` | 方向 PD 输出限幅 |
| `pid_kp:值` | `pid_kp:2.5` | PID 比例系数 |
| `pid_ki:值` | `pid_ki:1.1` | PID 积分系数 |
| `pid_kd:值` | `pid_kd:1.2` | PID 微分系数 |

### 持久化命令

| 命令 | 功能 |
|------|------|
| `WRITE` | 保存当前参数到 `/home/root/car_config.txt` |
| `READ` | 从文件重新加载参数 |

## 四、LADRC 调参顺序

| 顺序 | 参数 | 作用 | 调大 | 调小 | 经验 |
|------|------|------|------|------|------|
| 1 | b0 | 系统模型参数 | 输出减弱 | 输出增强 | 先设初值再微调 |
| 2 | wc | 控制器带宽 | 响应快(可能震荡) | 响应平滑 | 从低往高，震荡后回调20% |
| 3 | w0 | 观测器带宽 | 扰动估计快 | 滤波强 | w0 ≈ 3~5×wc |
| 4 | r | 跟踪速度 | 跟踪快 | 更平滑 | 不宜过大 |
| 5 | h | 积分步长 | — | — | 通常不改 |

## 七、VSCode 远程连接

- 龙芯 2K0300 是 LoongArch 架构，VSCode Remote-SSH **不支持**
- 替代方案：
  - **sshfs** 挂载小车目录到 PC 本地编辑
  - scp 手动同步代码
  - SSH + vi/nano 板载编辑

```bash
# 挂载

# 强制卸载
```

## 八、图像传输

- Vofa+ **不支持显示图像**，只能看波形/数据
- 看摄像头画面需要用 **逐飞助手 (Seekfree Assistant)**
- 需要在 main.cpp 中加 `#include "my_image_transmitter.hpp"`，调用 `img_transmitter_init()` 和 `rgb_img_transmitter()`
- 修改 `my_image_transmitter.hpp` 中的 `SERVER_IP` 为 PC 实际 IP

## 1. 项目概览

这是一辆基于 **LS2K0300** 处理器的智能车，用于全国大学生智能车竞赛"走马观碑"组别。

| 功能 | 实现方式 |
|------|----------|
| 电机控制 | 双直流有刷电机（DRV驱动）+ 无刷电机（电调），支持 **PID / LADRC** 双控制方案 |
| 摄像头 | USB UVC 摄像头，320×240@60fps，**OpenCV** 处理 |
| IMU | 逐飞 IMU660RB/IMU963RA，**IIO sysfs** 驱动，**Madgwick AHRS** 姿态解算 |
| 显示屏 | IPS 200 (ST7789V)，240×320 SPI，**TTF 矢量字库** (MiSans，7000+汉字) |
| 人机交互 | 4 按键 (上/下/OK/返回) + 层级菜单系统（活编码） |
| 上位机 | **Vofa+** 协议实时调参 + **TCP** 远程参数读写 |
| 惯性导航 | 里程计 + 路径录制/复现 + **Akima** 插值 + 高斯滤波平滑 |
| 图像处理 | 迷宫巡线 + 透视变换 + Otsu 二值化 + 状态机(直道/十字/圆环) |
| AI 推理 | NCNN / TensorFlow Lite Micro (已集成，暂未启用) |
| 图像传输 | UDP 图像传输到上位机（调试用） |

## 4. 线程架构

所有线程基于 Linux `zf_driver_pit_rt` (POSIX `timer_create` 实时定时器) 实现，使用 `SIGEV_THREAD` 通知模式。

### 线程配置表

| 线程名 | 对象 | 周期 | 优先级 | 回调函数 | 功能 |
|--------|------|------|--------|----------|------|
| 编码器采集 | `encoder_get` | 2ms (`ENCODER_SAMPLING_PERIOD`) | **99** (最高) | `encoder_get_count_handler()` | 读编码器计数→速度映射→里程计更新 |
| PID方向控制 | `pid_control_thread` | 10ms (`5×2ms`) | **97** | `pid_contol_handle()` | 方向PD控制→`onto_control` 计算 |
| LADRC速度控制 | `lardc_control_thread` | 1ms (`LARDC_PERIOD`) | **99** (最高) | `hight_frequence_encoder_get_speed_handler()` | T/M法测速+LADRC速度闭环+差速PWM输出 |
| 按键+IMU | `key_scan` | 10ms (`KEY_SCAN_PERIOD`) | **95** | `key_scan_handler()` | 按键扫描+IMU读取+AHRS姿态解算 |

### 线程启动条件

在 `car_init()` 中根据 `control_model` 选择:

| `control_model` | 启动的线程 | 说明 |
|-----------------|-----------|------|
| `0` (PID模式) | `encoder_get` (pri 99, 2ms) + `pid_control_thread` (pri 97, 10ms) | PID速度闭环 |
| `1` (LADRC模式, **默认**) | `lardc_control_thread` (pri 99, 1ms) + `pid_control_thread` (pri 97, 10ms) | LADRC速度闭环 + 方向PD |

无论哪种模式，`key_scan` 线程 (pri 95, 10ms) 始终启动。

> ⚠️ **注意**: 当前 `main.cpp` 中 `car_init()` 被注释，仅启动了 `key_scan` 线程。控制线程需要手动或通过菜单功能触发。

### 周期宏定义 (my_global.hpp)

```cpp
#define ENCODER_SAMPLING_PERIOD  2    // 编码器采样周期 (ms)
#define LARDC_PERIOD             1    // LADRC 控制周期 (ms)
#define KEY_SCAN_PERIOD          10   // 按键扫描周期 (ms)
#define PID_CONTROL_PERIOD       (5 * ENCODER_SAMPLING_PERIOD)  // = 10ms
```

### 6.3 线程回调函数详解

#### `key_scan_handler()` — 10ms, pri 95

```cpp
void key_scan_handler() {
    key_manager.scan_keys();                            // 按键状态机扫描
    imu963r.update();                                   // IMU原始数据读取+低通滤波
    ahrs.updateIMU(                                     // Madgwick姿态解算（6轴），含坐标系重映射:
        gyro[0] * IMU_calibration,                      //   gx = gyro_x × 180/154.03
        gyro[2] * IMU_calibration,                      //   gy = gyro_z (Z→Y)
       -gyro[1] * IMU_calibration,                      //   gz = -gyro_y (Y→-Z)
        acc[0],  acc[2], -acc[1]                        //   加速度同样重映射
    );
    // 路径录制/复现（当前被注释）
    // if(path_tracker_component.is_recording)
    //     path_tracker_component.record_sample(ahrs.getYaw());
}
```

#### `encoder_get_count_handler()` — 2ms, pri 99

```cpp
void encoder_get_count_handler() {
    get_and_remap_speed(&right_speed, &left_speed, 2);  // 脉冲→速度映射，清零编码器
    // 里程计更新（录制/复现时）
    if(path_tracker_component.is_recording || path_tracker_component.is_reproduction) {
        path_tracker_component.right_tyre.update((int16_t)right_speed);
        path_tracker_component.left_tyre.update((int16_t)left_speed);
    }
}
```

#### `pid_contol_handle()` — 10ms, pri 97

```cpp
void pid_contol_handle() {
    if(onto_pd_control_enable == 1) {
        // 当前使用正弦波测试信号（调试用）
        onto_control = pid_angle.compute(
            calculate_yaw_control(90.0f * (sin(ti_tick/20.0f) >= 0 ? 1.0f : -1.0f),
                                  ahrs.getYaw(), 25), 0);
    } else {
        onto_control = 0;
    }
}
```

> ⚠️ 注意: 当前方向控制使用正弦波测试信号，实际巡线的 `onto` 变量（来自图像处理）被注释掉。

#### `hight_frequence_encoder_get_speed_handler()` — 1ms, pri 99

```cpp
void hight_frequence_encoder_get_speed_handler() {
    lardc_get_speed(1, 1024, 0.035, 0.001, &left_speed, &right_speed);  // T/M法测速
    if(onto_pd_control_enable == 1) {
        target_speed_l = target_speed_r = cruising_speed;                // 基准速度
        speed_to_pwm_l = ladrc_left.calculatePWM(target_speed_l + onto_control, left_speed);
        speed_to_pwm_r = ladrc_right.calculatePWM(target_speed_r - onto_control, right_speed);
    }
    motor_set_speed_ladrc(speed_to_pwm_l, speed_to_pwm_r);              // PWM输出
}
```

**差速转向公式**: 左轮 = 基准速度 + onto_control，右轮 = 基准速度 - onto_control

### 7.2 菜单核心机制

**双模式运作** (`mode_inter_flag`):

| 模式 | `mode_inter_flag` | 行为 |

## 12. LADRC 控制器

**文件**: `project/code/LADRC.cpp` (258行) / `LADRC.hpp` (201行)

### 12.1 一阶 LADRC 算法 (速度环)

**1. 跟踪微分器 (TD)**:
```
dv1/dt = v2
dv2/dt = -r²(v1 - v0) - 2r·v2
```
对期望速度 v0 进行平滑跟踪，v1 为平滑后位置，v2 为微分。r 越大跟踪越快。

**2. 一阶扩展状态观测器 (LESO)**:
```
dz1/dt = z2 - β₁·e + b₀·u       (z1 跟踪实际速度)
dz2/dt = -β₂·e                   (z2 跟踪总扰动)
β₁ = 2·w0,  β₂ = w0²
```
其中 `e = z1 - y` (y 为实际反馈值)，w0 为观测器带宽。

**3. 线性反馈律 (LSEF)**:
```
u₀ = Kp·(v1 - z1)     (Kp = wc, 一阶)
u = (u₀ - z2) / b₀    (扰动补偿)
```

### 12.2 预设参数 (5组)

| 预设 | h | r | wc | w0 | b0 | 场景 |
|------|---|----|----|------|-----|------|
| 0 | 0.005 | 20 | 100 | 400 | 0.5 | 快速响应 |
| 1 | 0.001 | 20 | 33 | 133 | 8.0 | **代码默认** |
| 2 | 0.005 | 100 | 20 | 80 | 0.5 | 高抗扰 |
| 3 | 0.005 | 100 | 14 | 57 | 0.5 | 强滤波 |
| 4 | 0.005 | 100 | 50 | 10 | 1.0 | 高带宽 |

**单向增量搜索** (`find_closest_index`):
- 从 `last_closest_idx` 开始向前搜索，窗口大小 `FORWARD_WINDOW=200`
- 不回头（倒车时索引锁死），O(k) 复杂度

**预瞄点** (`get_look_ahead_point`):
- 取最近点 + `look_ahead_dist_idx` (默认 3) 作为预瞄目标点
- 通过 `calculate_target_yaw()` 计算期望航向: `atan2(dy, dx)`

### 14.3 AkimaInterpolator (Akima 插值)

**文件**: `project/code/akima.cpp` (148行) / `akima.hpp` (74行)

Akima 分段立方插值:
- 至少需要 5 个点
- 自动计算各段斜率权重的加权平均
- 边界外推虚拟斜率

**地图保存流程** (`get_map` 菜单中):
```
原始路径点 → 高斯滤波(σ=1.2, k=5)
    → 计算累计里程 s_list
    → Akima x_engine.compute(s, x), y_engine.compute(s, y)
    → save_as_tracking_map(等距采样 resolution=10) → tracking_map.txt
    → convert_txt_to_bin() → tracking_map.bin (直接 fread 到内存)
```

**bin 文件格式**: 连续 `MapPoint{int32_t index; float x; float y;}` 结构体 (12字节/点)

## 18. 控制数据流

### 18.1 速度控制闭环 (LADRC模式)

```
target_speed_l/r ──→ SimpleMotorLADRC.calculatePWM() ──→ motor_set_speed_ladrc()
        ↑                                                        │
        │                                                        ▼
        │                                                  PWM输出 → 电机
        │                                                        │
        └──── lardc_get_speed() ←── 编码器脉冲 ←─────────────────┘
              (T/M累积法, 齿轮比30:68, 轮径0.035m)
```

### 18.2 方向控制

```
摄像头 ──→ image_proc() ──→ onto (角度偏移) ──→ pid_angle.compute() ──→ onto_control
                                                                              │
                                          ┌───────────────────────────────────┤
                                          ▼                                   ▼
                              target_speed_l += onto_control    target_speed_r -= onto_control
                                          │                                   │
                                          └──── 差速转向 ─────────────────────┘
```

### 18.3 惯性导航数据流

```
编码器 ──→ lardc_get_speed() ──→ 左/右轮速度(m/s)
        ──→ Odometer.update(脉冲) ──→ 里程累积
                                          │
IMU ──→ ahrs.getYaw() ───────────────────┤
                                          │
                        PathTracker.record_sample(yaw)
                                          │
                        path_array[] (x, y, yaw) [最多24000点]
                                          │
                        高斯滤波(σ=1.2, k=5) → Akima插值 → tracking_map.bin
```

### 18.4 路径复现数据流

```
tracking_map.bin ──→ load_binary_map() ──→ tracking_map vector
                                                  │
编码器+IMU ──→ get_location() ──→ current_location │
                                                  │
                        find_closest_index() ←────┘
                        (单向增量搜索, O(k))
                                                  │
                        get_look_ahead_point(3) ──→ target_point
                                                  │
                        calculate_target_yaw() ──→ 期望航向角
```

### 19.4 注意事项

1. **硬件反接**: `left_pwm` 控制右电机，`right_pwm` 控制左电机（两个 motor_set_speed 函数已内置交换）
2. **IIO 设备路径**: 默认 `iio:device1`，若实际编号不同需修改 10 个宏 (`zf_device_imu.hpp`)
3. **IMU 型号**: 当前为 DEV_IMU660RB (无磁力计)，Yaw 漂移不可避。若使用 IMU963RA 则有磁力计
4. **线程优先级**: 编码器/LADRC(99) > PID(97) > 按键+IMU(95)
5. **PWM 输出差异**: PID模式用 `motor_set_speed` (speed×60→PWM)，LADRC模式用 `motor_set_speed_ladrc` (pwm×10→PWM)
6. **IPS 显示性能**: `clear()` + 绘制 + `update()` 全过程约 30ms，频繁更新会严重打乱线程周期
7. **编码器采样周期**: >1000ms 会导致 int16 溢出
8. **齿轮减速比**: 电机端编码器经过 30:68 减速比到轮端，移植时需重新标定
9. **PWM 死区**: ±50，低于此值电机不转，移植时需根据实际电机调整
10. **`wait_image_refresh()` 返回值**: **0=成功, -1=失败**，不要用布尔判断
11. **配置文件路径**: `car_init()` 中硬编码为 `/home/root/car_config.txt`（嵌入式 Linux 根目录）
12. **菜单添加新功能**: 在 `menu_table[]` 添加条目→在 `my_task_function.cpp` 实现→套壳后挂载 handler
13. **新增源文件**: 需在 `libraries/zf_common/zf_common_headfile.hpp` 中注册
14. **TTF 字库**: 由 CMake 选项 `USE_TTF` 控制，`font_data.cpp` 由构建系统自动生成

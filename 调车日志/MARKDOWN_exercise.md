# Markdown 笔记书写教程

> 面向：嵌入式/智能车开发者
> 工具：任何文本编辑器（VSCode 推荐）

## 一、十分钟速成

Markdown 就是用纯文本写排版。下面是最常用的 10 种语法，覆盖 90% 的笔记场景。

### 标题

```markdown
# 一级标题（每篇笔记只用一次，当文章标题）
## 二级标题（大章节）
### 三级标题（小节）
#### 四级标题（更细的分点）
```

### 正文与换行

```markdown
直接写就是正文。连续两行之间空一行就分段。

这是第二段。两个空格+回车=强制换行（不常用）。
```

### 加粗、斜体、代码

```markdown
**加粗**          → **加粗**
*斜体*            → *斜体*
`行内代码`        → `行内代码`
~~删除线~~        → ~~删除线~~
```

### 列表

```markdown
- 无序列表项 1
- 无序列表项 2
  - 缩进两个空格就是子项

1. 有序列表项 1
2. 有序列表项 2
```

### 代码块

````markdown
```cpp
int main() {
    printf("hello\n");
    return 0;
}
```

```bash
ls -la
gcc main.c -o main
```

```json
{"key": "value"}
```
````

三个反引号后跟语言名，VSCode 就会语法高亮。常用：`cpp` `bash` `python` `json` `markdown`，不写也行。

### 链接与图片

```markdown
[链接文字](https://example.com)
![图片描述](path/to/image.png)
```

### 表格

```markdown
| 参数 | 值 | 说明 |
|------|-----|------|
| Kp | 2.5 | 比例系数 |
| Ki | 1.1 | 积分系数 |
```

### 分隔线

```markdown
---
```

````markdown

bool car_init() {
````

## 三、进阶技巧

### 引用块

```markdown
> 这是一个引用块。适合标注"注意"、"警告"、或者引用别人说的话。
```

效果：
> 这是一个引用块。

### 任务列表

```markdown
- [x] 已完成的任务
- [ ] 待完成的任务
- [ ] 另一个待办
```

效果：
- [x] 已完成的任务
- [ ] 待完成的任务

### 折叠细节（VSCode/GitHub 支持）

```markdown
<details>
<summary>点击展开详细日志</summary>

这里放大段的日志输出，默认折叠不占空间。

</details>
```

### 转义特殊字符

```markdown
如果想显示 `*` `#` `` ` `` 等符号本身而不是被解析，前面加 `\`：

\* 不是斜体 \*
\# 不是标题
\` 不是代码
```

### Emoji

```markdown
:white_check_mark: → :white_check_mark:
:x: → :x:
:warning: → :warning:
:bulb: → :bulb:
```

## 六、常见错误

| 错误写法 | 正确写法 | 说明 |
|----------|----------|------|
| `#标题` | `# 标题` | `#` 后必须有空格 |
| `` ```cpp `` 后不空行 | 正常 | 代码块开始和结束标记独占一行 |
| 表格 `|` 不对齐 | 没关系的 | 对齐只是为了源码可读，不影响渲染 |
| 列表项间不留空行 | 各有用途 | 不留空行=紧凑列表，留空行=段落间距 |
| 中文和英文/数字间没空格 | 建议加 | `PID 参数` 比 `PID参数` 好看 |

---

> 最后一条：Markdown 的设计哲学是**源码也要可读**。即使不渲染，纯文本也能看懂结构。别过度排版，内容第一。

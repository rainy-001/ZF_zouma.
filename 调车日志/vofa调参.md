## 1. 整体架构

```
┌─────────────────────┐          TCP (Port 8086)          ┌──────────────────────┐
│     PC (Vofa+)       │ ◄────────────────────────────── │   LS2K0300 (小车)     │
│     TCP Server        │                                 │   TCP Client          │
│     192.168.43.94     │     命令 →                       │                      │
│                       │     ← 数据                       │                      │
└─────────────────────┘                                  └──────────────────────┘
```

- **PC 端**：Vofa+ 作为 **TCP Server**，监听 8086 端口
- **车端**：LS2K0300 作为 **TCP Client**，主动连接 PC
- **通信**：双向 — PC 发调参命令，车可回传实时数据到波形图
- **网络**：车和 PC 必须在同一局域网（通常 PC 开 WiFi 热点，车连上来）

### 2.1 修改 `project/user/main.cpp`

            }

                handleReadCommand();              // 从文件重新加载参数

### 2.2 关键修改点

| 位置 | 说明 |
|------|------|
| `"192.168.43.94"` | **必须改成你 PC 的实际 IP 地址** |
| `onto_pd_control_enable = 1` | 使能方向控制（调参时建议开启） |
| `client.receiveLine(...)` | 非阻塞接收，100ms 超时 |
| `client.sendFormattedData(...)` | 可选，上传数据到 Vofa+ 波形图 |

```bash

```

```cmd

```

## 3. 第二步：PC 端启动 Vofa+ TCP Server

### 3.1 打开 Vofa+

1. 启动 Vofa+ 软件
2. 点击左上角**协议选择区域**（默认显示 "Serial Port" 或类似文字）
3. 在下拉菜单中选择 **TCP Server**

### 3.2 配置 TCP Server

| 参数 | 值 | 说明 |
|------|-----|------|
| 本地端口 | `8086` | 与车端代码端口一致 |
| IP 地址 | `0.0.0.0` | 监听所有网卡（默认即可） |

### 3.3 启动服务

点击 **启动** 按钮，Vofa+ 状态栏应显示 "等待连接..." 或类似提示。

### 3.4 确认连接

- 启动小车（编译并运行修改后的代码）
- 终端应打印 `Connected to 192.168.x.x:8086`
- Vofa+ 状态栏显示连接成功

## 5. 第四步：调参命令参考

### 5.1 实时调参命令（立即生效，不保存到文件）

命令通过 TCP 以纯文本发送，格式为 `参数名:值\n`。

#### LADRC 参数

| 命令 | 示例 | 范围建议 | 说明 |
|------|------|----------|------|
| `b0:值` | `b0:0.83` | 0.1 ~ 50 | 系统模型参数。越小→输出越强，越大→输出越弱 |
| `wc:值` | `wc:29` | 5 ~ 200 | 控制器带宽。越大→响应越快（可能震荡） |
| `w0:值` | `w0:190` | 20 ~ 500 | 观测器带宽。越大→扰动估计越快（噪声增大） |
| `r:值` | `r:13.3` | 5 ~ 500 | 跟踪速度因子。越大→跟踪越快 |
| `h:值` | `h:0.001` | 0.0005 ~ 0.01 | 积分步长。通常不改，与采样周期对应 |

> **注意**：`b0`、`wc`、`w0`、`r`、`h` 修改后**左右轮同步更新**。

#### 方向环 PD 参数

| 命令 | 示例 | 说明 |
|------|------|------|
| `kp:值` | `kp:0.02` | 线性比例系数（基础修正力） |
| `kp2:值` | `kp2:0.0167` | 非线性比例系数（大偏差时附加修正） |
| `kd:值` | `kd:1.9` | 微分系数（抑制震荡，预测趋势） |
| `limit:值` | `limit:2` | 输出限幅（差速最大值） |

#### PID 模式参数（仅在 control_model=0 时有效）

| 命令 | 示例 | 说明 |
|------|------|------|
| `pid_kp:值` | `pid_kp:2.5` | PID 比例系数 |
| `pid_ki:值` | `pid_ki:1.1` | PID 积分系数 |
| `pid_kd:值` | `pid_kd:1.2` | PID 微分系数 |

### 5.2 持久化命令

| 命令 | 功能 |
|------|------|
| `WRITE` | 将当前内存中所有参数**保存**到 `/home/root/car_config.txt` |
| `READ` | 从配置文件**重新加载**参数到内存（恢复上次 WRITE 保存的值） |

### 5.3 参数生效流程

```
Vofa+ 发送 "wc:35"
       │
       ▼
TCP 传输到车端
       │
       ▼
parseAndUpdateParameter("wc:35")
  ├── lardc_r_wc = 35.0
  ├── lardc_l_wc = 35.0
  └── printf("更新 wc: 35.000000\n")
       │
       ▼
param_setting()  自动调用
  ├── ladrc_right.init(h, r, wc, w0, b0, ...)
  └── ladrc_left.init(h, r, wc, w0, b0, ...)
       │
       ▼
下一个 LADRC 控制周期 (1ms) 即使用新参数
```

> 参数修改后**无需重启**，在下一个控制周期自动生效。

## 6. 第五步：调参工作流

### 6.1 推荐流程

```
┌─────────────────────────────────────────────────────────┐
│  1. 让小车跑起来（低速），观察行为                        │
│       │                                                  │
│       ▼                                                  │
│  2. 在 Vofa+ 发送调参命令                               │
│       │   参数立即生效                                   │
│       ├── 满意 → 发送 WRITE 保存到文件                   │
│       └── 不满意 → 继续调                                │
│       │                                                  │
│       ▼                                                  │
│  3. 如果调乱了，发送 READ 恢复到上次保存的值              │
│       │                                                  │
│       ▼                                                  │
│  4. 重复 2-3，逐个参数收敛                               │
│       │                                                  │
│       ▼                                                  │
│  5. 最终确认后 WRITE 保存                                │
└─────────────────────────────────────────────────────────┘
```

### 6.2 LADRC 调参顺序与经验

| 顺序 | 参数 | 作用 | 调大效果 | 调小效果 | 经验法则 |
|------|------|------|----------|----------|----------|
| **1** | `b0` | 系统模型参数 | 输出减弱 | 输出增强 | 先凭经验设，再微调 |
| **2** | `wc` | 控制器带宽 | 响应更快，可能震荡 | 响应平滑 | 从低往高调，震荡后回调 20% |
| **3** | `w0` | 观测器带宽 | 扰动估计更快，噪声增大 | 滤波更强 | 通常 w0 ≈ 3~5 × wc |
| **4** | `r` | 跟踪速度 | 目标跟踪更快 | 更平滑 | 不宜过大，避免超调 |
| **5** | `h` | 积分步长 | — | — | 通常不改，与 1ms 控制周期对应 |

### 6.3 方向 PD 调参顺序

| 顺序 | 参数 | 作用 | 当前值 |
|------|------|------|--------|
| **1** | `kd` | 微分（抑制震荡） | 1.9 |
| **2** | `kp` | 线性比例（基础修正） | 0.02 |
| **3** | `kp2` | 非线性补偿（大偏差修正） | 0.0167 |
| **4** | `limit` | 输出限幅 | 2 |

> 方向环先调 `kd` 抑制震荡，再调 `kp` 确定响应力度，最后微调 `kp2` 和 `limit`。

### 6.4 调参技巧

1. **一次只改一个参数**，观察效果后再改下一个
2. **每次改动幅度**：wc/w0 每次 ±5~10，b0 每次 ±0.1~0.5
3. **震荡判断**：听电机声音（高频啸叫）、看速度波动（Vofa+ 波形图）
4. **快速恢复**：调乱了直接发 `READ`，回到上次保存状态
5. **阶梯测试**：用 Vofa+ 波形图观察阶跃响应（突然给速度，看超调量）

## 7. 第六步：实时数据可视化

### 7.1 启用数据上传

取消 `main.cpp` 中 `sendFormattedData` 的注释：

```cpp
client.sendFormattedData(
    "Onto_control,yaw,speed_l,speed_r,l_pwm,r_pwm:%f,%f,%f,%f,%d,%d\r\n",
    onto_control, ahrs.getYaw(),
    left_speed, right_speed,
    speed_to_pwm_l, speed_to_pwm_r);
```

### 7.2 在 Vofa+ 中添加波形图

1. 在 Vofa+ 主界面右键 → **添加控件** → **波形图**
2. 配置波形图的数据源为 TCP Server 接收端口
3. 波形图会自动解析 `key:value` 格式的数据

### 7.3 可监控的变量

| 变量 | 含义 | 调参用途 |
|------|------|----------|
| `onto_control` | 方向 PD 控制量 | 观察方向修正是否平滑 |
| `yaw` | 当前航向角 (°) | 观察转弯角度变化 |
| `speed_l` | 左轮实际速度 (m/s) | 观察速度闭环跟踪效果 |
| `speed_r` | 右轮实际速度 (m/s) | 同上 |
| `l_pwm` | 左轮 PWM 输出 | 观察控制量是否饱和 |
| `r_pwm` | 右轮 PWM 输出 | 同上 |

### 7.4 添加自定义监控变量

在 `sendFormattedData` 的格式串中追加即可：

```cpp
// 追加 cruising_speed 和 target_speed
client.sendFormattedData(
    "yaw,speed_l,speed_r,cruising:%f,%f,%f,%f\r\n",
    ahrs.getYaw(), left_speed, right_speed, cruising_speed);
```

## 8. 故障排查

### 8.1 TCP 连接失败

| 现象 | 可能原因 | 解决方法 |
|------|----------|----------|
| `Connection failed` | IP 地址不对 | 检查 PC 的 IP，确保和代码中一致 |
| `Connection failed` | Vofa+ 没启动 TCP Server | 确认 Vofa+ 已点"启动" |
| `Connection failed` | 不在同一网络 | PC 开热点，车连热点 |
| `Connection failed` | 防火墙拦截 | 关闭 PC 防火墙或放行 8086 端口 |
| 连接成功但收不到命令 | 主循环没进 receiveLine | 确认 main.cpp 中相关代码已取消注释 |

### 8.2 调参不生效

| 现象 | 可能原因 | 解决方法 |
|------|----------|----------|
| 发命令没反应 | 发送格式不对 | 确保末尾有换行符 `\n`，格式如 `wc:35` |
| 参数变了但车没反应 | 控制线程没启动 | `car_init()` 被注释掉了，需要手动启动控制线程 |
| 参数变了但左右不一致 | `parseAndUpdateParameter` 同步逻辑 | 确认是内置同步的参数（b0/wc/w0/r/h 会自动同步） |

### 8.3 通信中断

| 现象 | 可能原因 | 解决方法 |
|------|----------|----------|
| 跑着跑着断了 | WiFi 信号不稳定 | 缩短距离，避免金属遮挡 |
| 收到乱码 | 分包/粘包 | `receiveLine` 以 `\n` 为分隔，发送时确保每行以 `\n` 结尾 |
| 高频率发命令时丢包 | TCP 缓冲区满 | 降低发送频率（Vofa+ 中调整 loop_ms） |

预设了 6 个快捷命令按钮：

| 按钮名称 | 发送内容 (HEX) | 解码 |
|----------|---------------|------|

| `adjust w0` | `77 30 3A 25 66 0A` | `w0:%f\n` |
| `adjust wc` | `77 63 3A 25 0A` | `wc:%\n` ⚠️ 有 bug，缺 `f` |
| `adjust r` | `72 3A 25 66` | `r:%f` |
| `write in` | `57 52 49 54 45 0A` | `WRITE\n` |
| `read` | `52 45 41 44 0A` | `READ\n` |

> ⚠️ `adjust wc` 的 HEX 缺少 `66`(f)，实际发送的是 `wc:%\n` 而非 `wc:%f\n`。如需使用请手动修正，或直接手动输入 `wc:值`。

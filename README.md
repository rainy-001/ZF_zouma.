# 21 智能车走马观碑组

> 此代码为 21 届走马组准备，因作者精力有限已无力维护，现做开源处理。  
> 本项目基于逐飞科技开源底层开发，并对其底层库（`ips`、`uvc`、`tcp` 等）进行了大量优化与改进。  
> 视觉代码参考上海交通大学开源方案并进行针对性设计与改造。

## ✨ 主要特性

- **控制算法**：实现了 PID、平方项 PD、LADRC 等多种控制方案。
- **上位机调参**：适配 `vofa+` 上位机通信协议，支持实时调参。
- **菜单系统**：具备完备的菜单结构与机制，按键映射灵活。
- **IMU 姿态解算**：陀螺仪解算代码来自 [x-io Technologies](https://x-io.co.uk/open-source-imu-and-ahrs-algorithms/)。
- **惯性导航探索**：已实现并验证了地图路径录制功能。
- **字体优化**：导入经剪枝的 MiSans 开源字库（支持 7000+ 汉字及所有英文字符），替代静态字库。

## 📘 使用说明

### 🔤 字库配置（TTF / 静态字库切换）

是否启用矢量字库由 CMake 选项控制：

- 配置文件：`project/user/CMakeLists.txt`
- 开关选项：`option(USE_TTF "Whether to use Vector Fonts" ON)`
- 字体文件路径：`set(FONT_INPUT_FILE "${CMAKE_CURRENT_LIST_DIR}/../../assets/fonts/MiSans_Mini.ttf")`

如需更换字体，修改上述路径即可。

### 🧵 线程管理

- 线程回调函数声明在 `project/code/my_global.cpp` 中，该头文件已导入全局文件。
- 如需新增自定义文件，请在 `libraries/zf_common/zf_common_headfile.hpp` 中进行注册。

### 📋 菜单任务挂载

- 菜单任务函数定义于 `project/code/my_task_function.cpp`，在此处实现具体功能性函数。
- 将函数通过模板套壳后挂载到菜单结构数组中即可，详见 `project/code/my_menu.cpp` 中的 `Menu MyMenu::menu_table[]` 及现有套壳函数示例。
- 套壳函数中已实现按键映射逻辑。

### ⚙️ 全局初始化

- 全局初始化函数 `car_init()` 位于 `my_global.cpp`。
- 该函数负责线程与组件的初始化，**建议先初始化组件，再启动线程**。
- 初始化过程会读取板卡本地的 `car_config.txt` 配置文件，根据内容选择控制模式并启动相应进程（配置文件已提供在工程中）。

### 🎛️ 参数设置与上位机调参

- 调参功能集中在 `project/code/config_seting.hpp`。
- 使用 `vofa+` 上位机进行实时调参，工程内已包含上位机配置文件。
- 如需增删调参对象，修改 `config_seting.hpp` 中的内容即可，**注意同步调整并添加缓存容器**。

### 🧩 功能性文件说明

除上述核心文件外，其余为功能性模块：

| 文件 | 说明 |
|------|------|
| `project/code/LADRC.cpp` | LADRC 算法核心实现 |
| `project/code/my_pid.hpp` | PID / LADRC 二次封装，统一接口 |
| `project/code/motor.hpp` | 电机相关函数，**注意**：PID 和 LADRC 设置占空比、获取编码器值的专用函数不同，请查看线程函数中的具体用法 |

### 🖼️ 图像处理修改指南

图像底层原子操作移植自上海交通大学开源方案，主要处理流程如下：

1. 主处理函数：`image_proc()`
2. 边线提取：`void line_process(uint8_t height_start, uint8_t height_min)` 获取左右边线、最大弯曲度（角点）及角点索引。
3. 状态机调度：`element_status()` 判断当前元素状态。
4. 各状态处理函数：
   - `no_element_process()`
   - `crossing_process()`
   - `circle_process()`
   - `auto_tracking()`

**改进建议**：
- 优先修改状态机逻辑及对应状态处理函数，非必要不修改底层算法。
- **⚠️ 特别注意**：当前用于从图像获取控制方向的 `calculate_weighted_offset_angle` 函数存在已知缺陷（权重调节繁琐、易引入震荡），**强烈不建议使用**，需后续修正。  
  （该函数原本可用，因工程回溯时误退版本，后续将修复）

### 🧰 其他注意事项

- 部分功能组件（如 `IPS200` 显示屏、`AHRS` 姿态解算器）使用了类的析构函数，实例化时需传入相关参数。
- 功能组件对象大多定义在 `project/code/my_global.hpp` 中，部分为便于跨文件引用在源文件中定义并 `extern` 导出，使用前请留意变量声明位置。
